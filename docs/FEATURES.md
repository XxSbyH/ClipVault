# ClipVault 功能文档

本文档面向 ClipVault 用户，说明当前项目已经实现的主要功能和边界。开发计划、迁移规格和验证记录仍保留在 `docs/superpowers/` 下，不作为用户功能手册使用。

## 核心工作流

ClipVault 运行后会在本机监听系统剪贴板，把符合规则的内容写入本地历史库。用户可以通过主面板搜索、预览、复制、收藏、删除或清理历史记录，也可以通过全局快捷键快速调用常用操作。

默认行为：

- 文本、URL、代码片段、颜色值、邮箱等文本类内容会按类型识别后保存。
- 图片会按配置压缩后保存，并在详情区域显示预览和尺寸信息。
- 文件类内容只保存路径和元数据，不复制文件内容本身。
- 重复内容会去重；已有文本如果后续读到富格式数据，会补充保存富格式信息。
- 所有历史、设置、黑名单和快捷键配置默认保存在本地，不提供云同步。

## 主面板

主面板是 ClipVault 的主要操作入口：

- 列表展示剪贴板历史，并支持大列表虚拟滚动。
- 搜索框支持普通文本搜索，也支持 `/pattern/i` 和 `re:pattern` 正则查询。
- 详情区域显示选中项的完整内容、图片预览、元数据、时间和使用次数。
- 键盘可使用 `ArrowUp` / `ArrowDown` 移动选择，`Enter` 复制选中项，`Esc` 隐藏面板。
- 选中项可以收藏或删除；收藏项会在清理历史时保留。

## 历史项操作与文本工具

历史列表支持对单条记录执行常用操作：

- 右键文本类历史项可以打开操作菜单。
- 特殊粘贴支持全部大写、全部小写、纯文本、驼峰命名、首字母大写、句首字母大写、移除换行、追加换行、追加当前时间。
- 文本工作台可以编辑当前历史内容，也可以把编辑后的文本另存为新记录。
- 文本类历史项可以直接添加为固定内容，后续绑定独立快捷键使用。
- 历史项可以删除；排序相关入口当前仍处于开发中，不作为已完成功能。

## 历史快速复制

默认全局快捷键：

- `Ctrl+Shift+V`：打开或隐藏主面板。
- `Ctrl+Shift+F`：打开主面板并聚焦搜索框。
- `Ctrl+Shift+P`：暂停或恢复剪贴板监听。
- `Ctrl+Shift+C`：清空非收藏历史记录。
- `Ctrl+Alt+Left`：复制更旧的历史内容到系统剪贴板。
- `Ctrl+Alt+Right`：复制更新的历史内容到系统剪贴板。

历史快速复制只把目标历史项写入系统剪贴板，不自动模拟粘贴。需要粘贴时，用户在目标应用里执行正常粘贴操作即可。

## 固定内容快捷键

设置面板的“快捷键”页支持维护固定内容：

- 为常用文本设置标题、内容和独立快捷键。
- 启用后按快捷键可直接写入剪贴板并粘贴。
- 保存时会检查与全局快捷键和其它固定内容快捷键的冲突。
- 可以随时编辑、停用或删除固定内容。

## 鼠标滚轮快捷键

设置面板可启用滚轮浏览历史：

- 按住指定修饰键并滚动滚轮，在历史记录中向更旧或更新方向切换。
- 可以选择触发修饰键和作用范围。
- 设置更新会先验证运行时 hook 能注册成功，避免只保存配置但功能不可用。

## 隐私过滤

敏感内容过滤默认启用，用于跳过常见高风险内容：

- 疑似信用卡号。
- 美国 SSN 类格式。
- 中国身份证号。
- `password=...` 或类似密码赋值。
- 长大写字母数字 token / API key 类内容。

命中的内容不会写入历史记录。过滤规则是本地规则，不会把剪贴板内容上传到外部服务。

## 应用黑名单

应用黑名单用于跳过指定应用里的剪贴板内容：

- 支持按应用名和规范化路径匹配，大小写不敏感。
- 内置黑名单用于覆盖常见密码管理器等敏感来源。
- 用户可以在设置面板添加或移除自定义黑名单应用。
- 从黑名单应用复制内容后，即使马上切换到非黑名单应用，同一次 Windows 剪贴板变更也不会被补录进历史。

最后一条行为依赖 Windows 剪贴板序列号。ClipVault 会把黑名单来源对应的剪贴板序列短期记入忽略列表，从而避免“复制时在黑名单应用，读取时已经切到普通应用”的误记录。

## 存储与清理

设置面板提供存储相关配置：

- 文本大小限制。
- 图片压缩策略。
- 历史保留天数。
- 最大历史条数。
- 清空非收藏历史。

自动清理会保留收藏项，并按保留天数和最大条数清理非收藏旧项。

## 导入与导出

设置面板的“存储”页支持历史备份：

- 导出 ClipVault 历史记录。
- 导入之前导出的历史数据。
- 导入导出范围包含历史记录和富格式数据。
- 不包含设置、应用黑名单或快捷键配置。

## 桌面集成

ClipVault 面向 Windows 桌面环境集成：

- 系统托盘菜单。
- 全局快捷键。
- 自启动设置。
- 单实例运行，第二实例会聚焦已有窗口。
- HUD 轻量反馈，用于提示复制成功、面板状态、快捷键动作和监听状态变化。
- 本地启动和运行日志，用于排查启动无窗口、WebView2、热键、托盘或剪贴板监听问题。

## 当前边界

ClipVault 当前不实现以下能力：

- 云同步。
- OCR。
- 团队协作。
- 视频播放。
- 复制文件内容本身。
- Electron fallback 或 Node sidecar。

文件类型只保存路径和元数据；如果原文件被移动或删除，ClipVault 不会持有文件副本。
