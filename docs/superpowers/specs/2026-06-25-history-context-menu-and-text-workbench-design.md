# History Context Menu And Text Workbench Design

## 背景

ClipVault 当前已经具备历史记录列表、详情预览、复制、粘贴、置顶、收藏、删除、固定快捷内容和快捷键设置能力。本设计在现有 Tauri 2 + React 架构上增强历史项操作，目标是让用户能更明确地选择、粘贴、转换、编辑和沉淀常用文本，同时避免把 ClipVault 扩展成文档编辑器或富文本管理器。

本轮设计已和 xxsby 确认以下边界：

- 历史项单击不再复制，改为选中并显示详情。
- 历史项双击直接粘贴到前台应用。
- 历史项右键菜单不放“复制”和“直接粘贴”两个一级项。
- 转换类动作放入“特殊粘贴”子菜单。
- 编辑器先做到 Level 2 文本工作台。
- 不处理 Ditto 式多剪贴板格式保存，不做 HTML/RTF/Markdown 富格式恢复。
- 固定快捷内容默认提供 3 条试用示例，不默认占用全局快捷键。
- 排序功能先保留灰显入口并标注“开发中”，首版不实现持久排序。

## 目标

1. 历史列表交互更接近成熟剪贴板工具：单击选择，双击粘贴。
2. 右键菜单提供高频历史项操作，但保持层级清晰。
3. 支持文本类历史项的特殊粘贴转换。
4. 提供 Level 2 文本工作台，用于粘贴前编辑和清理文本。
5. 在设置里展示 3 条固定快捷内容试用示例，引导用户创建真实固定内容。
6. 允许从历史项快速预填固定内容表单。

## 非目标

- 不保存或恢复 Windows 剪贴板里的 HTML Format、Rich Text Format、CF_DIB、CF_HDROP 等原始多格式组合。
- 不实现富文本编辑器、Markdown 预览、代码高亮编辑器、插件系统或模板变量系统。
- 不在首版实现历史项自定义排序、拖拽排序或排序数据库迁移。
- 不改变文件项“只保存路径和元数据”的隐私边界。
- 不新增云同步、团队协作或 OCR。

## 方案选择

### 方案 A：最小改动

只修改单击/双击行为，新增右键特殊粘贴和固定内容入口，编辑内容使用普通 textarea 弹窗。

优点是改动少、风险低。缺点是 Level 2 文本工作台无法形成完整体验，后续还要重新设计编辑区、文本转换复用和保存语义。

### 方案 B：推荐方案

一次性完成历史项交互、右键菜单、特殊粘贴、Level 2 文本工作台、固定内容示例和从历史项预填固定内容表单。排序菜单保留灰显“开发中”入口。

优点是首版功能闭环清晰，特殊粘贴和编辑器可以共享文本转换能力。缺点是涉及前端、Tauri command、SQLite 更新和测试，实施范围中等。

### 方案 C：完整能力方案

同时实现持久排序、富格式保存、完整文档编辑器和固定内容管理增强。

优点是能力完整。缺点是明显偏离本地剪贴板管理器主线，增加兼容性、数据库迁移和桌面粘贴风险。

最终采用方案 B。

## 用户体验设计

### 历史列表交互

- 单击历史项：只选中该项，并更新右侧详情预览。
- 双击历史项：调用粘贴命令，写入系统剪贴板，隐藏主窗口，向前台应用模拟 `Ctrl+V`。
- `Enter`：直接粘贴当前选中项。
- `Ctrl+C`：复制当前选中项到系统剪贴板，不隐藏主窗口。
- `Delete`：删除当前选中项。
- `Ctrl+D`：收藏或取消收藏当前选中项。
- `Ctrl+P`：置顶或取消置顶当前选中项。

取消单击复制后，不需要通过延迟判断区分单击和双击，列表选择会更跟手，也避免用户只是查看、右键或编辑时意外覆盖系统剪贴板。

### 右键菜单

历史项右键菜单首版包含：

- 特殊粘贴
- 编辑内容
- 剪切项排序（开发中）
- 添加为固定内容
- 删除

菜单不包含“复制”和“直接粘贴”两个一级项。复制保留在详情按钮、键盘快捷键或未来更明确的位置。直接粘贴由双击和 `Enter` 承担。

“剪切项排序（开发中）”作为灰显子菜单保留，子项包括：

- 上移（开发中）
- 下移（开发中）
- 固定到顶部（开发中）
- 移动到底部（开发中）

首版不改变仓库排序逻辑，不新增排序字段。

## 特殊粘贴

### 适用范围

特殊粘贴只对文本类历史项启用：

- `text`
- `url`
- `code`
- `color`
- `email`

图片和文件项不显示特殊粘贴，或显示为灰显不可用。文件项继续只保存路径和元数据，不参与文本转换。

### 转换动作

特殊粘贴子菜单包含：

- 全部大写：将文本转换为 uppercase 后粘贴。
- 全部小写：将文本转换为 lowercase 后粘贴。
- 仅粘贴纯文本：按当前保存的纯文本内容粘贴，不保留任何未来可能支持的富格式。
- 驼峰命名法：将文本转换为 lower camel case，例如 `hello world` 到 `helloWorld`。
- 首字母大写：只将整段文本的第一个字母大写。
- 句首字母大写：将每个句子的首字母大写。
- 移除换行符：将连续换行压成空格，保留词间可读性。
- 粘贴并添加一个换行符：在文本尾部追加一个换行符。
- 粘贴并添加显示当前时间：在文本尾部追加当前本地时间。

### 执行模型

前端传递历史项 id 和转换动作给 Tauri command。后端读取历史项，确认是文本类内容，执行转换，写入系统剪贴板，隐藏主窗口并模拟 `Ctrl+V`。转换后的文本不自动写回历史记录，除非用户在文本工作台里选择保存。

执行成功后更新该历史项使用统计并发送 history revision。执行失败时返回清晰错误，不更新使用统计。

## Level 2 文本工作台

### 适用范围

文本工作台只允许编辑文本类历史项。图片和文件项的详情预览保留现有只读行为，不进入文本编辑器。

### 功能

文本工作台提供：

- 原始内容和编辑结果区域。
- 直接编辑文本。
- 查找替换。
- 撤销和重做。
- 大小写转换。
- 驼峰命名法转换。
- 首字母大写和句首字母大写。
- 移除换行符。
- 尾部追加换行符。
- 尾部追加当前本地时间。
- 保存到当前历史项。
- 另存为新历史项。
- 添加为固定内容。

文本工作台不提供：

- 富文本格式工具栏。
- Markdown 预览。
- HTML 或 RTF 保存。
- 代码高亮编辑。
- 模板变量和插件。

### 保存语义

保存到当前历史项时：

- 更新 `content`。
- 更新 `preview`。
- 重新计算 `content_hash`，避免后续去重状态错误。
- 保留 `content_type`、收藏、置顶、使用统计和创建时间。
- 清空或更新不再适用的文本 metadata。
- 触发 FTS 更新。
- 发送 history revision。

另存为新历史项时：

- 创建新的文本类历史项。
- 走现有隐私过滤和文本长度限制。
- 若内容与已有历史项 hash 相同，复用现有去重语义。
- 新项进入历史列表顶部，选中新项。

添加为固定内容时：

- 打开固定内容表单。
- 预填标题和内容。
- 不立即创建固定内容。
- 用户录制快捷键并保存后，才创建真实固定内容。

### 标题和预览

编辑器标题使用历史项预览的前若干字符作为初始标题提示。固定内容预填标题从文本首行生成：

- 优先取首行非空内容。
- 过长时截断。
- 空内容不允许保存。

## 固定快捷内容试用示例

设置页的固定快捷内容区域默认展示 3 条试用示例。它们不是数据库中的真实固定内容，不占用快捷键，不参与冲突检测，也不注册全局快捷键。

示例建议：

1. 常用回复：`收到，我稍后处理。`
2. 邮件签名：`谢谢，祝好。`
3. 日期占位：`今天需要同步的事项：`

每条示例显示简短提示，例如“点击使用后可录制快捷键并保存”。用户点击示例后打开固定内容表单并预填标题和内容，快捷键为空，状态默认启用，保存前必须录制快捷键。

若用户已经创建真实固定内容，示例仍可显示在“试用示例”折叠区或次级区域，避免和真实列表混淆。

## 前端设计

### 组件职责

- `ClipboardList`：处理虚拟列表、选择、双击粘贴、键盘行为和右键菜单入口。
- `ClipboardItem`：渲染历史项卡片，单击只触发选择，双击触发粘贴。
- 新增 `ClipboardItemContextMenu`：渲染右键菜单和特殊粘贴子菜单。
- 新增 `TextWorkbenchDialog` 或等价组件：渲染 Level 2 文本工作台。
- `SettingsPanel`：增加固定内容试用示例，并支持外部预填打开固定内容表单。
- `tauriApi`：增加特殊粘贴、更新历史项、另存为历史项等 API 映射。
- `clipboardStore`：保持选中项和列表更新逻辑，接收后端返回的更新项。

### UI 风格

保持当前 teal/orange 视觉语言、Lucide 图标和键盘优先体验。右键菜单和编辑器控件要紧凑，不使用营销式大卡片。所有按钮和菜单项需要有可见 hover 和 focus 状态。

## 后端设计

### Tauri commands

新增或扩展 commands：

- `special_paste_item(id, action)`：对文本类历史项执行特殊粘贴。
- `update_text_item(id, content)`：保存编辑结果到当前历史项。
- `create_text_item(content)`：另存为新的文本历史项。

现有 commands 保持稳定：

- `paste_item`
- `copy_item`
- `delete_item`
- `toggle_pin`
- `toggle_favorite`
- `list_fixed_contents`
- `create_fixed_content`
- `update_fixed_content`
- `delete_fixed_content`

### 文本转换模块

新增共享文本转换模块，供特殊粘贴和文本工作台复用。转换函数应是纯函数，便于单元测试。

建议转换动作枚举：

- `upper`
- `lower`
- `plain`
- `camel`
- `capitalize`
- `sentence`
- `removeNewlines`
- `appendNewline`
- `appendCurrentTime`

时间追加使用本地时间，格式建议为 `YYYY-MM-DD HH:mm:ss`。该格式对用户可读，不依赖地区化字符串。

### Repository

新增仓库方法：

- 根据 id 更新文本历史项内容、预览、hash 和 metadata。
- 插入新的文本历史项。

更新文本项时要确保 FTS 触发器继续工作。现有 `clipboard_items_au` 触发器会在 UPDATE 后重建对应 FTS 行。

## 错误处理

- 特殊粘贴用于非文本项：返回 `unsupported content type for special paste`。
- 历史项不存在：返回 `clipboard item {id} not found`。
- 文本内容为空且动作需要内容：返回清晰错误，不粘贴。
- 写剪贴板失败：返回错误，不更新使用统计。
- 模拟粘贴失败：返回错误，不更新使用统计。
- 保存编辑结果为空：前端阻止保存，后端也校验。
- 更新文本项产生 hash 冲突：采用明确策略，优先拒绝并提示“内容已存在”，避免破坏唯一索引。

## 测试策略

### Rust 单元测试

- 特殊粘贴拒绝非文本项。
- 特殊粘贴对每个转换动作产生正确文本。
- 特殊粘贴成功后更新使用统计和 revision。
- 特殊粘贴失败不更新使用统计。
- 更新文本历史项会更新 content、preview 和 hash。
- 更新文本历史项拒绝空内容。
- 更新文本历史项 hash 冲突返回清晰错误。
- 另存为文本历史项遵守插入和去重语义。

### 前端测试

- 单击历史项只选择，不调用 `copyItem`。
- 双击历史项调用 `pasteItem`。
- `Enter` 调用 `pasteItem`。
- `Ctrl+C` 调用 `copyItem`。
- 右键菜单对文本项显示特殊粘贴，对图片和文件项禁用或隐藏。
- 特殊粘贴菜单调用正确 API。
- 文本工作台能执行转换、查找替换、保存当前项和另存为新项。
- 从历史项添加为固定内容会预填设置表单。
- 设置页默认显示 3 条试用示例且不调用创建 API。

### 手工验证

需要真实 Windows 会话验证：

- 双击历史项能粘贴到记事本。
- `Enter` 能粘贴到前台应用。
- 特殊粘贴每个转换动作能写剪贴板并粘贴。
- 文本工作台保存后历史列表和搜索结果更新。
- 从历史项添加为固定内容后快捷键注册生效。
- 文件项仍然只保存路径和元数据。
- 敏感内容过滤和应用黑名单行为不被绕过。

## 风险与缓解

- 双击粘贴可能改变现有用户习惯：详情区域保留明确复制按钮，键盘提供 `Ctrl+C`。
- 文本工作台范围扩大：严格限制为 Level 2，不引入富文本和插件。
- 特殊粘贴和编辑器转换重复：抽出共享纯函数模块。
- 更新历史项可能影响 hash 去重：后端重新计算 hash，并对冲突返回清晰错误。
- 右键菜单与虚拟列表事件冲突：菜单状态放在列表层，使用 item id 定位，不依赖 DOM 持久存在。

## 验收标准

- 历史项单击只选择，不复制。
- 历史项双击和 `Enter` 能直接粘贴。
- 右键菜单结构符合设计，不包含“复制”和“直接粘贴”一级项。
- 文本类历史项支持特殊粘贴转换。
- 图片和文件项不会执行特殊文本转换。
- Level 2 文本工作台可编辑并保存文本历史项。
- 固定内容试用示例不会占用快捷键。
- 从历史项可预填固定内容表单。
- 排序功能以“开发中”形式展示，不改变排序结果。
